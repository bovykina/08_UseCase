# Сценарии использования

## Описание  
В этом проекте я изучала работу с Use Case — как описывать взаимодействие пользователя и системы через сценарии. Поняла разницу между целями пользователя и его действиями, а также научилась описывать основные и альтернативные сценарии.

## Полученные навыки  
- Описание Use Case (UC)  
- Определение действующих лиц, их целей  
- Формирование предусловий, триггеров, постусловий  
- Описание основного и альтернативных потоков   

## Этапы проекта
1. Описание UC менеджера для работы с расписанием  
2. Описание основного и альтернативных сценариев
3. Описание UC клиента для бронирования услуг
4. Описание UC  мастера для просмотра своего расписания и отзывов

## Заключение  
Проект помог понять, что с помощью Use Case можно легко описать логику работы отдельных функций системы.
